# 国版 政策シミュレーター 画面検討メモ

## 1. 画面設計の前提

既存の静的Webアプリ構成を活かし、MVPでは複雑なルーティングを追加しない。

画面は `src/app.js` の `activeView` による切り替えを継続する。既存の学園版画面を国版へ置き換える。

```text
dashboard -> ダッシュボード
voices    -> 声の分析
issues    -> 政策ターゲット
policy    -> 政策案・分析チャット
result    -> 実行結果
save      -> 保存モーダル
```

MVPでは、最初から画面数を増やしすぎない。政策ターゲット指定、声の分析、政策案、実行結果の4つを中心にし、保存とAI設定は既存モーダルを流用する。

## 2. ナビゲーション

### MVPナビ

- ダッシュボード
- 政策ターゲット
- 声の分析
- 政策案
- 実行結果
- 保存
- AI設定

### 表示順

```text
ダッシュボード
  ↓
政策ターゲット
  ↓
声の分析
  ↓
政策案
  ↓
実行結果
```

ただし、ユーザーは各画面を自由に戻って確認できる。実行後のみ、政策ターゲットと政策案の編集操作をロックする。

### 既存画面との対応

| 既存画面 | 国版MVP画面 | 変更内容 |
| --- | --- | --- |
| ダッシュボード | ダッシュボード | 学園指標を日本の政策指標へ差し替え |
| 声の分析 | 声の分析 | 生徒の声を国民・ステークホルダーの声へ差し替え |
| 課題設定 | 政策ターゲット | 課題候補選択を廃止し、政策指定へ置換 |
| 施策検討 | 政策案 | 政策案と政策分析チャットを統合 |
| 結果 | 実行結果 | 年度末ではなく単発政策実行結果へ置換 |
| 保存 | 保存 | IndexedDB / JSON保存を流用 |

## 3. ダッシュボード画面

### 目的

政策を決める前に、日本の初期状態とシミュレーション前提を把握する。

### 主要領域

1. ヘッダーサマリー
2. 社会・経済・財政指標
3. 国際関係スコア
4. 人口構成サマリー
5. データ出典・データ種別

### 表示項目

#### ヘッダーサマリー

- シナリオ名
- データ基準年
- 政策実行状態
  - 未選択
  - 分析中
  - 政策案作成済み
  - 実行済み
- 現在選択中の政策名

#### 指標カード

カテゴリ別に表示する。

- 社会指標
- 経済・財政指標
- 政策実行指標
- 国際関係指標

各カードに表示する。

- 指標名
- 値
- 単位
- データ種別
- 基準年
- 簡単な説明

データ種別は視覚的に区別する。

- 実統計値
- 実統計参考値
- 仮想指標
- AI推定

#### 国際関係スコア

横並びまたは小型テーブルで表示する。

- アメリカ
- EU
- 中国
- アジア
- グローバルサウス
- 国際機関

各クラスターに表示する。

- 関係スコア
- 経済依存度
- 安全保障感度
- サプライチェーン重要度
- 政策反応メモ

#### 人口構成サマリー

MVPではグラフを3つに絞る。

- 世代構成
- 所得階層
- 地域構成

政策効果分析の土台であることが分かるよう、声の分析や実行結果の世代別・所得別効果と同じセグメントIDを使う。

### 主な操作

- `AIで初期化`
- `政策ターゲットへ進む`
- `詳細データを見る`

### MVPで後回し

- 実統計値のオンライン更新
- 出典URLの詳細ビュー
- 指標の手動編集

## 4. 政策ターゲット画面

### 目的

シミュレーション対象の政策を選ぶ、または自由記述から作る。

### 主要領域

1. プリセット政策選択
2. 自由記述入力
3. 政策強度選択
4. AI構造化結果
5. 関連指標・想定影響のプレビュー

### プリセット政策

MVPでは3つ。

- 消費税減税
- 子育て支援給付の拡充
- 行政DXの推進

各プリセットには以下を表示する。

- 政策名
- 1行概要
- 主な対象
- 主な論点
- 財源注意ラベル

### 自由記述入力

入力欄は1つに絞る。

例。

```text
最低賃金を全国一律で大きく引き上げる政策を検討したい
```

### 政策強度

セグメントコントロールで選ぶ。

- 小規模
- 標準
- 大規模

### AI構造化結果

AIが返す `PolicyTarget` を表示する。

- 政策名
- 政策概要
- AI生成の政策分野
- 主対象
- 主要な受益者
- 主要な負担者
- 想定財源
- 実施期間
- 関連指標
- 主な論点
- 財源上の注意

### 主な操作

- `政策を分析する`
- `自由記述を政策ターゲット化`
- `この政策で進める`

### 状態

- 未選択
- プリセット選択中
- 自由記述入力中
- AI構造化中
- 政策ターゲット確定済み
- 実行後ロック

## 5. 声の分析画面

### 目的

政策ターゲットに対する国民・ステークホルダーの想定反応を確認する。

### 主要領域

1. 反応サマリー
2. クラスターマップ
3. 世代別・所得別の想定効果
4. 代表発話
5. 階層クラスタリング

### 反応サマリー

上部に4つの比率を表示する。

- 賛成
- 反対
- 条件付き
- 無関心

補助的に表示する。

- 便益が大きいクラスター数
- 不利益が出るクラスター数
- 反発が強いクラスター数
- 実務上の懸念数

### クラスターマップ

EChartsの散布図を使う。

軸はMVPでは固定ラベルにする。

- 横軸: 政策への賛否
- 縦軸: 利害の強さ

点の表現。

- サイズ: クラスター規模
- 色: stance
- 濃さ: 感情強度

クリック時に表示する。

- クラスター名
- 要約
- 代表発話
- 関連ペルソナ
- 関連指標

### 世代別・所得別の想定効果

声の分析画面では、実行前の期待効果として表示する。

#### 世代別

- 10代・20代
- 30代
- 40代
- 50代
- 60代
- 70代以上

#### 所得別

- 低所得層
- 中間層
- 高所得層

各セグメントに表示する。

- 効果スコア
- 便益レベル
- リスクレベル
- 1行理由
- 該当性

政策との関連が薄い場合は、効果スコアを空にし、`該当性が低い` と理由を表示する。

### 代表発話

カードで表示する。

- 発話者ラベル
- 複合ペルソナ
- stance
- mood
- 発話本文
- 関連指標
- 関連クラスター

特定の政治的立場、宗教・支持組織、社会運動に近いペルソナを表示する場合も、個人推定ではなく仮想反応軸として表示する。

## 6. 政策案画面

### 目的

AIが生成した政策案を確認し、チャットで調整して、実行前の最終案にする。

MVPでは「政策案」と「政策分析チャット」を1画面にまとめる。画面を分けると導線が長くなるため、左に政策案、右にチャットを置く。

### レイアウト

デスクトップ。

```text
┌──────────────────────────────┬──────────────────────┐
│ 政策案プレビュー              │ 政策分析チャット       │
│                              │                      │
│ 実施内容                      │ 質問入力              │
│ 財源案・警告                  │ AI回答履歴            │
│ 期待効果                      │                      │
│ 世代別・所得別効果            │                      │
│ クラスター別便益・不利益      │                      │
│ 国際関係への影響              │                      │
└──────────────────────────────┴──────────────────────┘
```

モバイル。

```text
政策案
↓
チャット
```

### 政策案プレビュー

表示する。

- 政策名
- 概要
- 対象者
- 実施手段
- 実施コスト
- 財源案
- 財源警告
- 短期効果
- 長期影響の予想
- 世代別の期待効果
- 所得別の期待効果
- 便益クラスター
- 不利益クラスター
- 反発クラスター
- 国際関係への影響
- 実装リスク
- 説明責任上の論点

### 政策分析チャット

初期の入力例をボタン化する。

- どの世代に効果が大きい？
- 所得別の不利益は？
- 財源を変えると反応は？
- 反発が強いクラスターは？
- 国際関係への影響は？
- 無関心層に届く説明は？

チャット回答には、参照したクラスターと指標を明示する。

### 主な操作

- `政策案を生成`
- `チャットで修正`
- `この内容で実行`

### 実行前チェック

実行ボタンの近くに表示する。

- 政策実行は1回のみ
- 実行後は政策案をロック
- 長期影響は予想であり確定結果ではない
- 財源制約は警告として扱う

## 7. 実行結果画面

### 目的

政策実行後の短期結果と長期予想を確認する。

### 主要領域

1. 実行結果サマリー
2. 短期指標変化
3. 世代別・所得別の短期効果
4. クラスター別影響
5. 国際関係クラスター別反応
6. 長期予想
7. 政策実行レポート

### 実行結果サマリー

上部に表示する。

- 政策名
- 実行日時
- 短期評価
- 主な改善
- 主な副作用
- 財源上の懸念

### 短期指標変化

増減カードまたは棒グラフで表示する。

- 家計可処分所得
- 税負担感
- 財政余力
- 社会保障持続性
- 政策納得度
- 行政現場負荷
- 国際信頼度

政策に関連する指標を優先表示し、関係が薄い指標は折りたたむ。

### 世代別・所得別の短期効果

声の分析画面と同じ `SegmentEffect` を使う。

表示。

- 効果スコア
- 便益レベル
- リスクレベル
- 理由
- `not_applicable` の場合は数値ではなく説明文

### クラスター別影響

テーブルまたはコンパクトカードで表示する。

- クラスター名
- 便益
- 不利益
- 納得度
- 支持変化
- 代表的な反応

### 国際関係クラスター別反応

表示する。

- アメリカ
- EU
- 中国
- アジア
- グローバルサウス
- 国際機関

各クラスターに表示する。

- 反応
- 関係スコア変化
- 通商・安全保障・サプライチェーンへの影響
- 外交説明コスト

### 長期予想

短期結果とは明確に分ける。

表示ラベルは `長期予想` とし、確定結果に見せない。

- 長期リスク
- 長期公共性への予想影響
- 長期的に観測すべき指標
- 別要因で変動しやすい点

### 主な操作

- `新しい政策で開始`
- `JSONエクスポート`
- `レポートをコピー`

## 9. UI密度と見た目の方針

国版政策シミュレーターは、業務系ダッシュボードに近い。

避けるもの。

- 大きすぎるヒーロー
- マーケティングLP風の見出し
- 装飾的なカード過多
- 単色に寄りすぎた配色
- 重要データがスクロール下に埋もれる構成

目指すもの。

- 密度はやや高め
- 情報はカテゴリ別に整理
- 政策判断に必要な情報を初期表示
- チャートは説明より比較を優先
- 操作ボタンは少なく、次に進む導線を明確にする
- 国際関係、世代別、所得別などの比較軸は視覚的に並べる

### 配色

既存の深緑系を残してよいが、国版では以下を追加する。

- 社会指標: 青系
- 経済・財政: 緑系
- リスク・反発: 赤系
- 注意・財源懸念: 黄系
- 国際関係: 紫に寄りすぎない紺・シアン系

### コンポーネント

- 指標カード
- セグメント効果バー
- クラスター散布図
- 代表発話カード
- 政策案プレビュー
- 警告パネル
- チャットパネル
- 実行結果レポート

```

```

`issues` は互換性のため一時的に残してもよいが、画面ラベルは `政策ターゲット` にする。

### 初期実装の優先順

1. ナビと画面ラベルを国版へ変更
2. 政策ターゲット画面の静的UIを作る
3. ダッシュボードの指標カテゴリを国版へ変更
4. 声の分析に世代別・所得別効果を追加
5. 政策案画面にチャットと実行前チェックを統合
6. 実行結果画面を単発実行向けに変更
7. 保存ファイル名とDB名を国版へ変更

## 11. MVP画面完了条件

- ユーザーが迷わず `政策ターゲット -> 声の分析 -> 政策案 -> 実行結果` に進める
- 政策実行が1回だけであることが画面上で分かる
- 世代別・所得別効果が、声の分析、政策案、実行結果の3箇所で一貫して見える
- 世代や所得との関連が薄い政策では、無理な数値ではなく該当性が低い理由が出る
- 国際関係クラスターの反応がダッシュボードと実行結果で見える
- 財源上の懸念が政策案と実行結果で見える
- 長期影響が確定結果ではなく予想として表示される
