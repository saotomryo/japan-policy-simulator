# 国版 政策シミュレーター 開発計画

## 1. 開発方針

まずはモックデータで国版の画面導線を実際に動かす。

AI連携、実統計値の精査、JSON Schemaの厳密化は重要だが、先に画面上で `政策ターゲット -> 声の分析 -> 政策案 -> 実行結果` の流れを触れる状態にする。画面で確認しながら、指標・効果軸・クラスター表示の粒度を詰める。

### 優先すること

- ローカルで起動してすぐ確認できること
- 固定モックデータでも一連の体験が成立すること
- 指標や効果軸を固定表示にせず、政策ごとの関連指標・推奨ビューに切り替えられること
- 小さく実装し、毎フェーズでブラウザ確認すること
- 既存の保存、AI設定、Provider設計を壊さず段階的に置き換えること

### 後回しにすること

- 実統計値のオンライン更新
- 本格的なAIプロンプト調整
- 政策分野分類の固定化
- 複数ターン運営
- 財源制約による強制失敗
- 長期結果の確定シミュレーション

## 2. MVP到達イメージ

MVPでは、ユーザーが以下をローカルブラウザで操作できる状態を目指す。

1. 国版ダッシュボードを見る
2. プリセット政策を選ぶ
3. 政策ターゲットの構造化結果を見る
4. 関連指標・推奨ビューが政策に応じて切り替わる
5. 国民・ステークホルダーの声を見る
6. 世代別、所得別、地域別、国際関係などの効果軸を切り替える
7. 政策案を見る
8. チャット風UIで分析観点を切り替える
9. 政策を1回だけ実行する
10. 短期結果と長期予想を分けて見る
11. 保存、JSONエクスポート、JSONインポートができる

## 4. 初期実装の最小チケット

まず着手する順番。

1. `data/scenarios/national/dashboard.json` を作る
2. `src/app.js` の読み込み先を国版データへ切り替える
3. ナビラベルを国版にする
4. 政策ターゲット画面を作る
5. 国版ダッシュボードを表示する

7. 政策案と実行結果を固定サンプルで表示する
8. ブラウザで一連の画面遷移を確認する

この段階ではAIをつながない。まず画面と状態遷移を触って確認する。

## 5. ブラウザ確認の標準手順

各フェーズの最後に行う。

```bash
npm run check
npm run dev
```

ブラウザで確認する。

```text
http://127.0.0.1:4173/
```

確認項目。

- コンソールエラーがない
- 主要画面に遷移できる
- デスクトップ幅で横スクロールがない
- モバイル幅で主要テキストがはみ出さない
- クリック操作で状態が変わる
- 保存・エクスポート対象に不要な秘密情報が入らない

## 7. リスクと対策

### 学園版の概念が残る

対策: まず画面文言とデータキーを国版に寄せる。内部互換のために一時的に `issues` を残す場合も、画面には出さない。

### データ構造を一気に変えすぎて壊れる

対策: Phase 1では互換キーを残して動かし、Phase 4でスキーマと保存形式を整理する。

### AI連携前に仕様が固まりすぎる

対策: sample Providerで操作感を確認し、効果軸や表示粒度を画面で詰めてからAIプロンプトに反映する。

### 指標表示が政策に合わない

対策: `relatedMetricIds` と `recommendedViews` を中核にし、政策ごとに初期表示を変える。

### 実統計値の扱いが重くなる

対策: MVPでは固定初期値に限定する。出典・基準年・データ種別を持たせ、オンライン更新は後回しにする。
